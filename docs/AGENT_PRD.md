# 五环共创培训智能体 PRD

## 1. 产品概述

### 1.1 产品名称

五环共创培训智能体

### 1.2 产品定位

面向培训场景的多角色、多智能体协同平台。系统以“总教练 Agent”为统一入口，调度深学、跟练、反思、共创、转化五个专业教练模块，覆盖培训前、中、后全流程。

平台把学员在学习过程中的批注、即时贴、问答、案例、共识报告和转化成果结构化沉淀为知识库，再用于生成个性化方案和后续培训复用。

### 1.3 产品目标

- 为学员提供结构化学习、反思、共创和训后转化支持。
- 为教师/班主任提供班级过程管理、优秀内容筛选、共创组织和成果追踪能力。
- 为管理员提供账号权限、模型配置、知识库和系统运行管理能力。
- 通过知识库沉淀，把单次培训成果转化为可复用的组织资产。

## 2. 用户与角色

### 2.1 学员

主要目标：

- 提交学习批注、即时贴、问答、反思案例、共创观点和转化成果。
- 查看自己的学习进度、优秀内容、共识报告和个人行动计划。
- 使用模型能力生成或优化个人学习产出。

核心权限：

- 登录学员端。
- 查看个人仪表盘。
- 提交学习内容。
- 点赞和互动。
- 查看公开优秀内容和共识报告。
- 配置个人模型服务。

### 2.2 教师 / 班主任

主要目标：

- 管理班级培训过程。
- 查看学员提交内容。
- 审核或查看 AI 合规判断结果。
- 发起共创主题、分组、投票和报告生成。
- 跟踪学员训后转化成果。

核心权限：

- 登录教师端。
- 查看班级仪表盘。
- 查看所有学员提交内容。
- 管理共创主题、分组和投票。
- 查看优秀内容池和知识库入库情况。
- 生成共识报告。
- 配置个人模型服务。

### 2.3 管理员

主要目标：

- 管理系统账号、角色权限、模型配置和知识库。
- 查看系统整体运行状态。
- 控制模块开关、入库规则和数据治理规则。

核心权限：

- 登录管理员端。
- 管理账号与角色。
- 查看全局数据。
- 管理知识库。
- 管理模型配置入口。
- 查看所有模块运行状态。

## 3. 核心业务流程

### 3.1 登录与权限流程

1. 用户输入账号和密码。
2. 系统校验账号身份。
3. 根据角色进入不同页面：
   - 学员进入个人学习工作台。
   - 教师进入班级运营工作台。
   - 管理员进入系统治理工作台。
4. 前端根据角色控制页面导航、功能按钮和可见数据。
5. 后端接口根据角色校验数据访问权限。

### 3.2 学习内容沉淀流程

1. 学员在对应模块提交内容。
2. 系统进行 AI 初步合规判断。
3. 合规内容进入班级内容池。
4. 同学可阅读、互动和点赞。
5. 点赞数超过参训人数 1/3 的内容进入优秀内容池。
6. 优秀内容结构化、标签化写入对应知识库。
7. 总知识库聚合各模块高价值内容，用于个性化方案生成。

### 3.3 共创流程

1. 教师创建共创主题和分组。
2. 学员围绕主题提交观点。
3. 系统对观点进行标签化、语义去重、合并和分类。
4. 系统进行多轮观点收集。
5. 当新增观点少于等于 3 条时，判断“智慧已饱和”，可提前结束收集。
6. 系统生成投票卡片。
7. 学员投票。
8. 系统统计票数，提取高排名观点。
9. 系统生成《共识报告》。
10. 共识报告和高价值观点进入共创知识库。

### 3.4 训后转化流程

1. 总教练生成《个性化方案建议报告》。
2. 学员确认后形成“我的方案”。
3. 转化教练生成《个人行动计划》。
4. 系统在训后 1 个月、3 个月等节点提醒学员提交转化成果。
5. 学员上传成果或案例。
6. AI 对成果物做初步评估。
7. 合格转化案例进入转化知识库。
8. 下一期培训生成方案时可引用历史转化案例。

## 4. 用户交付逻辑优化

### 4.1 设计原则

产品首页不应只展示“有哪些模块”，而应优先回答用户进入系统后的三个问题：

- 我现在要完成什么任务。
- 完成后会交付什么成果。
- 这个成果交给谁，以及下一步由谁接手。

因此产品按角色建立交付闭环：

- 学员端强调“我的学习产出到转化成果”。
- 教师端强调“班级过程管理到方案交付”。
- 管理员端强调“系统配置到知识资产治理”。

### 4.2 学员交付路径

学员端交付路径为：

1. 完成学习证据。
   - 目标：围绕课程、实训和反思提交高质量学习产出。
   - 交付物：批注、即时贴、反思案例。
   - 接收方：班主任与同学。
2. 参与班级共创。
   - 目标：提交可被讨论、去重、投票的观点。
   - 交付物：共创观点。
   - 接收方：共创教练 Agent。
3. 接收个人方案。
   - 目标：查看总教练生成并下发的个人行动计划。
   - 交付物：我的方案、行动计划。
   - 接收方：学员本人。
4. 回传转化成果。
   - 目标：在训后节点提交真实应用证据。
   - 交付物：转化成果、转化案例。
   - 接收方：转化知识库。

### 4.3 教师交付路径

教师端交付路径为：

1. 收齐班级产出。
   - 目标：检查学员提交、AI 初审和互动数据。
   - 交付物：班级内容池。
   - 接收方：班主任。
2. 沉淀优秀素材。
   - 目标：推动点赞达标内容进入知识库。
   - 交付物：优秀批注、即时贴、问答。
   - 接收方：五环知识库。
3. 完成群体共识。
   - 目标：运行观点收敛、投票统计和报告生成。
   - 交付物：《共识报告》。
   - 接收方：全班学员。
4. 交付训后方案。
   - 目标：调用总知识库生成个性化方案并跟踪转化。
   - 交付物：方案建议报告、追踪清单。
   - 接收方：学员与转化教练。

### 4.4 管理员交付路径

管理员端交付路径为：

1. 配置进入规则。
   - 目标：确认账号、角色、导航和操作权限。
   - 交付物：账号权限清单。
   - 接收方：系统运营方。
2. 监控模块运行。
   - 目标：查看五个教练模块提交量、入库率和异常状态。
   - 交付物：模块运行看板。
   - 接收方：管理员。
3. 治理知识资产。
   - 目标：检查来源、标签、入库质量和复用价值。
   - 交付物：结构化知识库。
   - 接收方：总教练 Agent。
4. 维护模型能力。
   - 目标：配置可用模型，保障审核和生成链路可用。
   - 交付物：模型调用配置。
   - 接收方：全系统。

### 4.5 页面优化要求

- 每个角色首页必须展示专属交付路径。
- 每个交付步骤必须包含状态、目标、交付物、接收方和入口。
- 交付路径中的入口应直接跳转到对应业务模块。
- 状态文案应结合真实数据变化，例如“已提交”“待下发”“已沉淀”“待收敛”。
- 功能模块仍保留，但优先级低于角色交付路径。
- 教师和管理员首页必须提供班级提交内容池，可按学员、环节和入库状态筛选。

### 4.6 五环提交模板优化

每个环的提交界面必须根据交付内容单独设计，避免所有环共用“标题 + 内容”的泛表单。

| 环节 | 提交内容 | 页面模板 | 质量判断重点 |
| --- | --- | --- | --- |
| 深学环 | 学习批注 | 看到什么、为什么重要、如何迁移 | 有具体观察、有个人理解、有迁移动作 |
| 跟练环 | 照片 + 说明 / 即时贴 | 照片内容、现场发现、操作经验、下次复用 | 能还原现场、有操作动作、可复用 |
| 反思环 | GROW 反思案例 | Goal、Reality、Options、Will | 目标清晰、现状有证据、行动可执行 |
| 共创环 | 共创观点 | 建议、理由、落地条件；每行一条观点 | 一行一观点、可执行、有理由 |
| 转化环 | 转化成果 / 转化案例 | 应用场景、采取动作、结果证据、下一步改进 | 有应用场景、有结果证据、有改进动作 |

转化环需支持成果提交和成果池展示。教师或管理员可查看成果状态，学员可按行动计划节点提交转化案例。

### 4.7 图片附件与互动区优化

系统需提供图片上传接口，支持学员在跟练环、转化环及其他需要证据的提交场景上传图片附件。

图片上传要求：

- 支持 jpg、png、webp、gif。
- 单张图片不超过 5MB。
- 上传后返回图片 URL、文件名、类型和大小。
- 提交学习内容时，图片 URL 应作为附件写入该条内容。
- 互动区和教师/管理员内容池需展示图片缩略图。

互动区优化要求：

- 内容卡片展示作者、环节、类型、标题、正文、附件、AI 摘要、标签、点赞进度和入库状态。
- 教师和管理员可在班级提交内容池中按不同学员查看提交。
- 学员端仍可查看公开互动内容并参与点赞。

## 5. 功能需求

### 5.1 总教练 Agent

功能说明：

- 作为系统统一入口。
- 接收培训需求、学习数据和各模块产出。
- 调度五个专业教练模块。
- 调用总知识库生成《个性化方案建议报告》。
- 将学员确认后的“我的方案”下发给转化教练。
- 把优秀内容结构化、标签化存入知识库。

输入：

- 培训需求。
- 学习批注。
- 即时贴。
- 问答内容。
- 反思案例。
- 共创观点。
- 投票结果。
- 转化成果。

输出：

- 《个性化方案建议报告》。
- “我的方案”。
- 知识库入库记录。

验收标准：

- 能基于知识库内容生成个性化方案。
- 方案中应包含引用案例或历史素材。
- 能正确调度不同模块的数据。

### 5.2 深学教练 Agent

功能说明：

- 引导学员提交学习批注。
- 判断批注是否有效、过短、偏题或不合规。
- 支持同学阅读和点赞。
- 点赞超过 1/3 参训人数后进入优秀批注池。
- 优秀批注进入深学知识库。

输入：

- 课程主题。
- 学员批注。
- 点赞数据。

输出：

- 合规判断结果。
- 优秀批注。
- 深学知识库记录。

验收标准：

- 能识别空泛、过短、无关内容。
- 点赞达标后内容状态自动变为优秀。
- 教师端可查看学员提交的批注。

### 5.3 跟练教练 Agent

功能说明：

- 支持模拟实训过程记录。
- 学员可提交“照片 + 说明”形式的即时贴。
- 识别关键发现、操作经验和技能要点。
- 对即时贴进行合规判断。
- 支持点赞和优秀即时贴沉淀。

输入：

- 实训照片。
- 即时贴说明。
- 点赞数据。

输出：

- 合规判断结果。
- 优秀即时贴。
- 跟练知识库记录。

验收标准：

- 学员端可上传即时贴。
- 教师端可查看学员上传内容。
- 点赞达标后进入优秀即时贴池。

### 5.4 反思教练 Agent

功能说明：

- 基于 GROW 模型设计“你问我答”流程。
- 引导学员从 Goal、Reality、Options、Will 四个维度反思。
- 对问题和回答分别进行合规判断。
- 支持互动、点赞和优秀问答沉淀。
- 提供反思案例模板。
- 检查案例完整性和合规性。

输入：

- 提问内容。
- 回答内容。
- 反思案例。
- 点赞数据。

输出：

- 优秀问答。
- 反思案例。
- 反思知识库记录。

验收标准：

- 能按 GROW 结构引导学员回答。
- 能识别回答是否具备深度。
- 合格反思案例可进入知识库。

### 5.5 共创教练 Agent

功能说明：

- 支持教师创建共创主题。
- 支持班级分组。
- 收集学员观点。
- 对观点自动打标、语义去重、合并和分类。
- 支持多轮收集和智慧饱和判断。
- 自动生成投票卡片。
- 统计票数并生成《共识报告》。
- 将高价值观点和共识报告写入共创知识库。

输入：

- 共创主题。
- 分组信息。
- 学员观点。
- 投票结果。

输出：

- 分类观点池。
- 投票卡片。
- 《共识报告》。
- 共创知识库记录。

验收标准：

- 新增观点少于等于 3 条时可自动终止循环。
- 能输出清晰的观点分类。
- 能根据投票结果生成共识报告。

### 5.6 转化教练 Agent

功能说明：

- 接收总教练下发的“我的方案”。
- 生成《个人行动计划》。
- 在训后 1 个月、3 个月等节点提醒学员。
- 支持上传转化成果或转化案例。
- AI 初步评估成果完成度、落地性和改进空间。
- 合格转化案例进入转化知识库。

输入：

- 我的方案。
- 转化成果。
- 转化案例。

输出：

- 《个人行动计划》。
- 节点提醒。
- AI 初评结果。
- 转化知识库记录。

验收标准：

- 能生成结构化行动计划。
- 能在指定节点创建提醒。
- 能对成果物给出初步评估。

## 6. 模型配置需求

### 6.1 配置目标

系统不固定绑定单一模型。用户可在前端自行配置 OpenAI-compatible 模型服务，包括 MiniMax、OpenAI、通义、DeepSeek 或其他兼容服务。

### 6.2 配置项

- 是否启用外部模型。
- API Base URL。
- 模型名称。
- API Key。
- 调用状态测试。

### 6.3 安全要求

- 默认不在项目环境变量中保存用户 API Key。
- API Key 不写入源码仓库。
- 前端本地配置仅用于当前用户调用。
- 后端日志不得输出 API Key。
- 错误信息不得回显完整 API Key。

## 7. 数据与知识库设计

### 7.1 内容对象字段

每条学习内容至少包含：

- 内容 ID。
- 模块类型。
- 提交人 ID。
- 提交人角色。
- 标题。
- 正文。
- 附件地址。
- 标签。
- AI 合规结果。
- AI 评估摘要。
- 点赞数。
- 是否优秀。
- 是否入库。
- 创建时间。
- 更新时间。

### 7.2 知识库分类

- 总知识库。
- 深学知识库。
- 跟练知识库。
- 反思知识库。
- 共创知识库。
- 转化知识库。

### 7.3 入库规则

- 内容必须通过合规判断。
- 批注、即时贴、问答和观点需满足点赞阈值。
- 点赞阈值默认为参训人数的 1/3。
- 案例类内容需满足完整性检查。
- 教师或管理员可查看入库记录。

## 10. 非功能需求

### 10.1 可用性

- 页面结构应按角色区分，避免不同角色看到相同工作台。
- 关键操作需要明确状态反馈。
- 上传、点赞、生成报告等操作应有成功或失败提示。

### 10.2 性能

- 首屏加载应控制在可接受范围内。
- 内容列表应支持分页或筛选。
- 模型调用应设置超时时间。

### 10.3 稳定性

- 外部模型失败时应使用本地规则兜底。
- 数据提交失败时应保留用户输入。
- 服务端接口应返回清晰错误码。

### 10.4 部署

- 支持本地运行和云端部署。
- 云端服务可运行在非 80 端口。
- 应提供健康检查接口。
- 应避免将密钥写入服务器环境变量或源码。

## 11. 验收标准

### 11.1 账号联动

- 使用学员账号提交内容后，教师端可查看。
- 教师端发起共创主题后，学员端可提交观点。
- 管理员端可查看角色与权限配置。

### 11.2 内容流转

- 批注、即时贴、问答、案例和观点可正确识别模块类型。
- 合规内容可进入内容池。
- 点赞超过 1/3 参训人数后进入优秀内容池。
- 优秀内容可进入对应知识库。

### 11.3 共创闭环

- 系统能完成观点收集、去重、分类、投票和报告生成。
- 新增观点少于等于 3 条时可判断智慧饱和。
- 共识报告能展示高排名观点和行动建议。

### 11.4 模型配置

- 用户可自行配置模型服务。
- 未配置模型时系统仍可使用本地规则运行。
- 模型调用失败时不影响基础提交流程。
- 系统不暴露或记录完整 API Key。

### 11.5 云端访问

- 云端地址可正常打开。
- CSS 和静态资源正常加载。
- 健康检查接口返回正常。
- 登录、提交、点赞、教师端查看等核心链路可用。

## 12. 里程碑建议

### M1：基础可用版本

- 登录和三类角色页面。
- 学员提交内容。
- 教师查看内容。
- 本地规则合规判断。
- 点赞与优秀内容池。

### M2：智能生成版本

- 模型配置。
- AI 合规判断。
- 个性化方案生成。
- 反思案例模板。
- 共识报告生成。

### M3：运营闭环版本

- 共创多轮循环。
- 智慧饱和判断。
- 训后提醒。
- 转化成果评估。
- 知识库检索和引用。

## 13. 风险与约束

- 外部模型质量不稳定，需保留本地规则兜底。
- 用户自填 API Key 存在泄露风险，需做好前端提示和日志脱敏。
- 知识库质量依赖入库规则，需避免低质量内容批量沉淀。
- 教师端必须能查看学员端提交内容，否则培训运营链路不成立。
- 云端部署需确保静态资源、服务端口和健康检查稳定。
