# 页面精简方案 v2

## 精简原则

1. **每页只做一件事** — 一个页面一个核心任务
2. **砍掉重复信息** — 仪表盘不需要看到所有数据，只需要看到"下一步做什么"
3. **按角色裁剪** — 不同角色看到完全不同的页面内容
4. **后台功能后置** — 管理员功能收敛到设置页，不散落在各处

---

## 页面结构

```
/login          → 登录页
/               → 仪表盘（按角色显示不同内容）
/ring/deep-study     → 深学环
/ring/practice       → 跟练环
/ring/reflection     → 反思环
/ring/co-creation    → 共创环
/ring/transformation → 转化环
/knowledge      → 知识库
/settings       → 设置（含模型配置、账号管理）
```

共 **8 个页面**（当前 9 个，去掉独立的"总教练"页和"模型配置"页，合并入仪表盘和设置）。

## 各页面内容分配

### 1. 登录页 `/login`

**保留：**
- 居中大标题 + 一句话说明
- 账号/密码输入 + 登录按钮
- 三个演示账号快速入口（小字，底部）

**删除：**
- 左侧大段介绍文案
- 账号卡片中的角色描述和冗余信息
- 左右分栏布局 → 改为居中单栏

---

### 2. 仪表盘 `/`

#### 学员看到：
**保留：**
- 我的提交统计（已提交 X 条，已入库 Y 条）
- 我的行动计划状态（有/无，确认/待确认）
- 3 个下一步操作按钮（提交批注 / 参与共创 / 查看方案）
- 到期的检查节点提醒（如有）

**删除：**
- 交付流程四步卡片
- 五个环节的统计卡片
- 公开优秀内容列表
- 各种 Metrics 指标网格

#### 班主任看到：
**保留：**
- 班级概览：提交人数/总人数，入库条数
- 共创状态：当前主题、观点数、是否已收敛
- 5 个环节入口（图标+名称+提交数，一行排开）
- 快速操作：选择学员生成方案

**删除：**
- 班主任工作台 Hero 区块的大段文字
- 交付闭环四步卡片
- 总教练中枢两列区域
- 共创收敛状态独立面板
- 班级提交内容池（移到各环节页）
- 今日运营队列

#### 管理员看到：
**保留：**
- 系统概览：总提交数、入库数、活跃学员数
- 5 个环节运行状态（入库率进度条）
- 账号管理入口 → 跳转到设置页

**删除：**
- 管理员数据看板（显性反馈/隐性反馈全套）
- 权限矩阵
- 交付治理链路四步卡片
- 小组参与热力
- 最新课堂反馈
- 关注队列
- 账号管理面板（移到设置页）

## 总结：砍掉了什么

1. **交付流程面板** — 所有角色的四步卡片删除（过于说教，用户不需要每次看到）
2. **总教练独立页** — 合并入仪表盘（选择学员 → 生成方案，一个按钮的事）
3. **管理员数据看板** — 删除显性/隐性反馈全套（过度设计，demo 不需要）
4. **权限矩阵** — 删除独立展示，只在账号管理表格中体现
5. **班级提交内容池（仪表盘上）** — 移到各环节页
6. **大量说明文字** — 每个环节的 purpose/does/delivery 说明全部精简为一行
7. **5 种彩色标签系统** — 统一为单色系
8. **图片上传区域** — 除了跟练环，其他环删除
